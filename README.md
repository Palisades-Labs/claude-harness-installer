# claude-harness-installer

Public bootstrap that decrypts the Palisades-Labs Claude Code harness's encrypted credentials file on a developer's machine. Two scripts, one job:

- `bootstrap.sh` — macOS + Linux
- `bootstrap.ps1` — Windows

This is the **decrypt-only** installer. It does NOT install Claude Code, register marketplaces, or fetch GitHub tokens. Those are handled by Claude Desktop (marketplace add + sync) and the user's existing Claude Code install. The installer's only job is to take the age-encrypted credentials file that Claude Desktop synced from the marketplace, prompt for the setup passphrase, decrypt it, and wire it into the user's shell.

## Who runs this

**Client employees.** Their admin sends them an install command (generated by `/client-admin:generate-installer`). They paste it into Terminal (Mac/Linux) or PowerShell (Windows) and enter the setup passphrase the admin sent them separately.

For the full step-by-step employee install walkthrough in plain English, see [EMPLOYEE-GUIDE.md](EMPLOYEE-GUIDE.md). Admins setting up the harness for their team should see [the admin guide in the marketplace-admin repo](https://github.com/Palisades-Labs/marketplace-admin/blob/main/ADMIN-GUIDE.md).

## Install command shape

The Mac/Linux command is a single line:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Palisades-Labs/claude-harness-installer/main/bootstrap.sh) --decrypt <org>/<repo>
```

The Windows command is four separate lines (do NOT collapse — chat clients soft-wrap and break the command):

```powershell
$env:DECRYPT_MODE='1'
$env:CLIENT_REPO='<org>/<repo>'
Set-ExecutionPolicy -Scope Process Bypass -Force
iwr -useb https://raw.githubusercontent.com/Palisades-Labs/claude-harness-installer/main/bootstrap.ps1 | iex
```

Partway through, the script prints:

```
[bootstrap] Setup passphrase:
```

That's the passphrase the admin sent through a separate channel (1Password, Slack DM, in-person). Type or paste it (no echo) and press Enter.

## Prerequisites

- **Claude Code** installed (the script does NOT install it).
- **Claude Desktop** installed and the client marketplace already added + synced (the encrypted credentials file is sourced from the synced marketplace directory).
- **`age`** for decryption — the script auto-installs it from GitHub releases to `~/.local/bin/age` if missing. No Homebrew or sudo required.

## What it does (idempotent)

Every step is safe to re-run.

1. Auto-installs `age` from GitHub releases if not on PATH (downloads to `~/.local/bin/age`).
2. Locates the marketplace-synced credentials file at `~/.claude/plugins/marketplaces/<marketplace-name>/credentials/credentials.env.age`. Marketplace name is derived from `<org>/<repo>` by stripping `-claude-harness` then `-harness` suffixes.
3. Fails with a clear "marketplace not synced yet" error if the file is missing — the fix is to wait for Claude Desktop to finish syncing, or remove + re-add the marketplace.
4. Prompts for the setup passphrase (read from `/dev/tty`, no echo, never appears in output).
5. Stores the passphrase securely — macOS Keychain (`palisades-labs-harness`), `~/.claude/credentials/.passphrase` chmod 600 on Linux, DPAPI on Windows.
6. Decrypts to `~/.claude/credentials/credentials.env` (chmod 600).
7. Adds a one-time `set -a; source …; set +a` stanza to your shell rc so future terminals auto-load the credentials as environment variables.
8. On macOS+bash, also wires `.bash_profile` to source `.bashrc` (login-shell quirk fix).

After install: open a NEW terminal window (so the rc changes take effect), then run `claude`.

## Platform support

- macOS (Apple Silicon + Intel) — `bootstrap.sh`
- Linux (Debian/Ubuntu, RHEL/Fedora) — `bootstrap.sh`
- Windows — `bootstrap.ps1`

## Troubleshooting

- **"claude: command not found" after install.** Open a new terminal window — the installer added env vars that only load in fresh shells.
- **"marketplace not synced yet" error.** Wait 30 seconds for Claude Desktop to finish syncing the marketplace, then re-run. If it still fails, remove and re-add the marketplace in Claude Desktop.
- **"Wrong passphrase — could not be decrypted."** Double-check the passphrase the admin sent (no extra spaces). If unclear, ask the admin to confirm.
- **Passphrase prompt seems frozen.** It's not frozen — it's waiting for input. Nothing appears on screen as you type/paste; that's intentional. Press Enter when done.
- **`age` install fails.** Check that the machine has internet access to GitHub releases. As a manual fallback, install `age` yourself from https://github.com/FiloSottile/age/releases and re-run.
- **`raw.githubusercontent.com` serves an old version of the script after a recent push.** GitHub's CDN can lag 5–30+ minutes. Use the commit-SHA-pinned URL (`/<sha>/bootstrap.sh` instead of `/main/bootstrap.sh`) until it catches up.

For deeper troubleshooting beyond this list (admin + maintainer scope), contact the harness maintainer.
